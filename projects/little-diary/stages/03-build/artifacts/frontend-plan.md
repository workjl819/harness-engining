# 前端计划

## 页面结构
- 单页 H5，顶部为项目介绍和新建按钮。
- 左侧或上方为日记列表，展示标题、更新时间和摘要。
- 右侧或下方为编辑区，支持新建、编辑、删除。
- 当没有日记时显示空状态提示。

## 状态流
- 页面初始化时从 `localStorage` 读取全部日记。
- 列表点击后把当前日记设为激活项，并同步到编辑区。
- 保存时根据是否存在激活项决定新增或更新。
- 删除后清空当前编辑状态，并重新渲染列表。

## 本地存储策略
- 使用单一键 `little-diary:entries` 保存日记数组。
- 每条日记包含 `id`、`title`、`content`、`createdAt`、`updatedAt`。
- 列表默认按 `updatedAt` 倒序展示。

## 技术实现
- 采用纯静态 `HTML + CSS + JavaScript`。
- 不引入前端框架，降低 demo 复杂度。
- 视觉上优先适配移动端，并保证桌面浏览也可正常使用。
